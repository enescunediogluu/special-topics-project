Check with: python --version


⚠️ KNOWN ISSUE & SOLUTION
═══════════════════════════════════════════════════════════════════════════════

ERROR: "HTTP error 404 while getting https://github.com/..."

This is a COMMON spaCy model download issue. The solution is simple:

QUICK FIX (Recommended):
  1. pip install -r requirements.txt
  2. python -m spacy download en_core_web_sm
  3. python demo_pipeline.py

That's it! The model downloads separately, which avoids the URL issue.


🎯 COMPLETE INSTALLATION (5 Minutes)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Navigate to Project Directory
  cd /path/to/skill-extraction-pipeline

STEP 2: Create Virtual Environment (Optional but Recommended)
  
  On Windows:
    python -m venv skill-env
    skill-env\Scripts\activate
  
  On macOS/Linux:
    python3 -m venv skill-env
    source skill-env/bin/activate

STEP 3: Install Base Packages
  pip install -r requirements.txt

STEP 4: Download spaCy Model (THIS IS THE KEY STEP)
  python -m spacy download en_core_web_sm

STEP 5: Test Installation
  python demo_pipeline.py

✓ Done! You should see the demo running with output.


❓ IF IT FAILS AT ANY STEP
═══════════════════════════════════════════════════════════════════════════════

Step 3 Error (requirements.txt):
  → Make sure requirements.txt is in current directory
  → Try: pip install --upgrade pip
  → Then: pip install -r requirements.txt

Step 4 Error (spaCy model):
  → Check internet connection
  → Try alternate method:
    pip install https://github.com/explosion/spacy-models/releases/download/en_core_web_sm-3.7.1/en_core_web_sm-3.7.1-py3-none-any.whl
  
  → If still fails, pipeline will use blank model (less accurate, still works)

Step 5 Fails (demo):
  → Read INSTALLATION_GUIDE.md for detailed troubleshooting
  → Run: python -c "import spacy; spacy.load('en_core_web_sm')"
  → Check errors and search for solution

💡 TIPS
═══════════════════════════════════════════════════════════════════════════════

TIP 1: Virtual Environment
  Always use a virtual environment to avoid dependency conflicts
  
  python -m venv skill-env
  source skill-env/bin/activate  # macOS/Linux
  skill-env\Scripts\activate     # Windows

TIP 2: Upgrade pip First
  pip install --upgrade pip setuptools wheel
  Then install requirements

TIP 3: Check Python Version
  python --version
  Should be 3.8 or higher

TIP 4: Network Issues?
  - If behind proxy, configure it
  - Check firewall settings
  - Try alternate spaCy installation method (see INSTALLATION_GUIDE.md)

TIP 5: Still Having Issues?
  1. Read INSTALLATION_GUIDE.md (detailed troubleshooting)
  2. Check the specific error against troubleshooting section
  3. Try the alternative installation methods


🚨 IF ALL ELSE FAILS
═══════════════════════════════════════════════════════════════════════════════

The pipeline can work WITHOUT the trained spaCy model (less accurate NER):

1. Install without model:
   pip install -r requirements.txt

2. The code will automatically fall back to a blank spaCy model

3. Everything still works, but NER accuracy is reduced

See INSTALLATION_GUIDE.md section "Using Without Full spaCy Model"
